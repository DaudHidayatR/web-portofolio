# DevSecOps & Backend Blog

This blog is the writing side of my personal portfolio. The goal is to document what I learn and build around **DevSecOps** and **Backend Engineering**.

## Main topics

I plan to focus on practical, project-backed writing:

- Secure SDLC practices that development teams can actually adopt.
- CI/CD security with SAST, dependency scanning, container scanning, and secret scanning.
- Backend security patterns for REST APIs, middleware, validation, and error handling.
- Kubernetes and OpenBao labs as supporting infrastructure for DevSecOps experiments.
- Linux security audit automation and deployment-readiness checks.

## Why this structure

The website should not be only a resume. Each article should connect a technical concept to a project, lab, or implementation decision. That gives visitors more than a list of tools: it shows how I think through secure delivery and backend reliability.

## Editorial direction

The first content series will follow the project strategy already documented in this repository:

1. Self-hosting a DevSecOps lab with Kubernetes, Istio, and OpenBao.
2. Building CI/CD security scanning workflows for Secure SDLC practice.
3. Writing secure REST API middleware patterns in Go.
4. Automating pre-deployment security reviews.
5. Turning Linux security audits into repeatable CI/CD checks.

Cloud tools may appear when they support the implementation, but the main positioning stays focused on DevSecOps and Backend.
